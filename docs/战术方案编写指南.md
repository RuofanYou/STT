# 战术方案编写指南

如果听不到播报、只能听到别人相关的播报，或实时战术板空白，先打开 **设置 → 战术方案 → 一键诊断**，按 3 步检查 TTS、当前方案命中和实时板显示范围。

设置页顶部的 **运行测试** 会按当前方案执行 `/st test` 全链；**停止测试** 会停止播报、实时战术板和屏幕提醒等测试输出。

> 💡 **听不到任何 TTS 声音？** 系统自带的中文 TTS（如 Windows 10 默认的 Huihui）某些机器上可能没装。加 QQ 群 **637144370** 找作者拿一份额外 TTS 语音包（Microsoft 自然神经语音 + 多个中文音色），双击安装就能用。

> 💡 GUI 里的核心设置项现在支持多段式 Tooltip：鼠标悬停看简要说明，按住 Alt 展开示例；Tooltip 里的 `{time}`、`{坦克}`、`pN` 等蓝色语法词可继续悬停查看词条详情。

> 💡 设置页字太小时，打开 **设置 → 系统 → 系统设置 → 设置面板字体大小**，可在 `0.90x` 到 `1.60x` 之间实时放大设置页字号和控件；窗口大小不跟随字号变化。

## 团队检查与耐久检查

团队检查仍是 Beta 功能，需要开启 **`/st debug`** 并 `/reload` 后才会在设置页显示。`/st bc me` 可以打开个人自检，`/st bc raid` 可以打开当前队伍监控；团队检查会覆盖食物、合剂、符文、武器油、团队增益和 STT 团员主动上报的耐久状态。

设置里的 **界面 → 耐久检查** 可以单独开启低耐久修理提醒和自动修理。低耐久提醒开启后，STT 会在装备最低耐久低于阈值时弹出一个小提示，也可以在战斗结束后提醒一次；自动修理开启后，访问可修理商人会直接修理装备，按住 Shift 与商人对话可临时跳过。

## 阵容设置助手（Beta）

阵容设置助手仍需要先开启 **`/st debug`** 并 `/reload`，入口在 **设置 → 团长工具 → 阵容设置助手**。

这里的 **源文本** 是唯一可编辑权威：别名和每个 BOSS 的 8 组阵容都写在同一个文本框里。任意位置写 `昵称 = 角色名-服务器` 都会被识别为别名，不再强制要求 `[人员]` 头；BOSS 阵容继续用 `[BOSS名]` 开块，每一行对应一个小队，竖线右侧是替补。

```
冰豆WST = 冰豆-白银之手
我就LR = 我就-白银之手

[M梦裂]
冰豆WST 豆豆DKT 千山FS | 我就LR
```

切换当前 BOSS 后，40 格网格会渲染该 BOSS 的主力名单。拖拽或编辑格子会回写源文本；右侧实时对照会显示已到、未到、未知昵称和多余团员。点 **应用到团队** 时仍复用原来的三阶段排小队流程，只是输入从当前 BOSS 文本解析而来。

飞书常用公式：

```
=TEXTJOIN(CHAR(10),TRUE,IF(A2:A50="","",A2:A50&" = "&B2:B50))
="[M梦裂]"&CHAR(10)&TEXTJOIN(CHAR(10),TRUE,IF(D2:D9="","",TEXTJOIN(" ",TRUE,D2:H9)&IF(I2:I9=""," "," | "&I2:I9)))
```


## 三条核心规则

**规则一：`{time:分:秒}` 决定什么时候播报**

```
{time:00:10} ...    ← 开战后第 10 秒
{time:01:30} ...    ← 开战后第 1 分 30 秒
{time:90}   ...     ← 开战后第 90 秒（和上面一样）
{time:00:30,p2} ... ← P2 开始后第 30 秒
```

每一行必须以 `{time:XX}` 开头，没有 `{time:XX}` 的行会被忽略。

如果你想按阶段相对时间播报，可以在时间后面加逗号和阶段标签：

```
{time:00:10,p1} {所有人}开场分散
{time:00:08,i1} {所有人}过渡准备
{time:00:20,p2} {所有人}P2 集合
{time:00:12,p1r2} {所有人}第二轮 P1 集合
```

如果某一行必须按绝对时间点准时触发，不继承设置里的“提前播报时间”，在时间后写 `,-0`：

```
{time:00:33,-0} {所有人}{@Announcer_1.ogg}
```

上面这一行会在开战后第 33 秒播放音效；即使玩家把全局提前播报设成 3 秒，也不会提前到 30 秒。



**规则二：`{谁}` 决定谁能听到**

```
{time:00:10} {所有人}全员分散
{time:00:20} {坦克}拉好仇恨
{time:00:30} {张三}你去打断
```

`{谁}` 后面紧跟的文字，就是那个人会听到的内容。

**规则三：`{谁}` 前面的文字是备注，不会播报**

```
{time:00:32} 亡者吐息1:{所有人}去左边头前
```

这里 `亡者吐息1:` 写在 `{所有人}` 前面，它只是给你自己看的备注标签，不会被念出来。
所有人只会听到"去左边头前"。

## `{谁}` 可以写什么？

### 全团

| 写法 | 效果 |
|------|------|
| `{所有人}` | 全团都听到 |

示例：`{time:00:10} {所有人}全员分散`

### Boss 技能（只看不播报）

| 写法 | 效果 |
|------|------|
| `{BOSS}` | 显示在时间轴面板，但不会播报给任何人 |
| `{boss}` | 同上 |

`{BOSS}` 比较特殊——写了它的行会在时间轴面板上显示为一个单元格，但不会匹配任何玩家，没人会听到语音播报。

用途：在面板上标出 Boss 技能释放时间，方便对照。

```
{time:00:07} {BOSS}{spell:1258610}     ← 面板上显示"青龙下凡"的技能图标和名称
{time:00:14} {BOSS}{spell:1262289}     ← 面板上显示"孤注一掷"
{time:00:14} {小德1}豹奔               ← 小德1 听到"豹奔"
```

前两行 `{BOSS}` 只在面板上显示（带技能图标），不播报；第三行 `{小德1}` 正常播。内置模板已自带 `{BOSS}`，你自己写方案时也可以手动加。

### 指定玩家

直接写角色名：

```
{time:00:20} {张三}你去打断
```

只有叫"张三"的玩家会听到"你去打断"，其他人什么都听不到。

### 坦克 / 治疗 / 输出

| 写法 | 谁听到 |
|------|--------|
| `{坦克}` | 所有坦克 |
| `{治疗}` | 所有治疗 |
| `{输出}` | 所有输出 |

示例：`{time:00:15} {治疗}准备刷血`

### 指定职业

| 写法 | 写法（中文） |
|------|------------|
| `{warrior}` | `{战士}` |
| `{paladin}` | `{圣骑士}` |
| `{hunter}` | `{猎人}` |
| `{rogue}` | `{潜行者}` |
| `{priest}` | `{牧师}` |
| `{deathknight}` | `{死亡骑士}` |
| `{shaman}` | `{萨满祭司}` |
| `{mage}` | `{法师}` |
| `{warlock}` | `{术士}` |
| `{monk}` | `{武僧}` |
| `{druid}` | `{德鲁伊}` |
| `{demonhunter}` | `{恶魔猎手}` |
| `{evoker}` | `{唤魔师}` |

示例：`{time:00:30} {法师}释放时光变速`

### 指定专精

比职业更精确。可以直接写专精名称：

| 写法 | 谁听到 |
|------|--------|
| `{戒律}` 或 `{discipline}` | 戒律牧师 |
| `{神圣}` 或 `{holy}` | 神圣牧师 **和** 神圣圣骑（两个都叫"神圣"） |
| `{恢复}` 或 `{restoration}` | 恢复德鲁伊 **和** 恢复萨满 |
| `{防护}` 或 `{protection}` | 防护战士 **和** 防护圣骑 |
| `{冰霜}` 或 `{frost}` | 冰霜法师 **和** 冰霜死骑 |

**所有 39 个专精都支持**，中英文均可。上面只列了容易重名的，其余如武器、狂怒、惩戒、奇袭、暗影、鲜血、元素、奥术、火焰、酒仙、踏风、平衡、浩劫、湮灭、恩护、增辉等同样可用。

**重名专精怎么区分？** 用加号组合职业名消歧：

```
{time:00:10} {神圣+牧师}开守护之魂    ← 仅神圣牧师，圣骑不受影响
{time:00:20} {神圣+圣骑士}圣佑术      ← 仅神圣圣骑
{time:00:30} {防护+战士}盾墙          ← 仅防护战士，不是防护骑
{time:00:40} {冰霜+法师}冰脉          ← 仅冰霜法师，不是冰霜死骑
{time:00:50} {治疗+牧师+戒律}开减伤    ← 三重条件：治疗职责 且 牧师职业 且 戒律专精
```

> **小贴士**：只写 `{神圣}` 不加职业名，神圣牧师和神圣圣骑都会听到——这不是 Bug，是设计。如果你想两个都喊，直接写 `{神圣}` 最省事。

### 近战 / 远程

| 写法 | 谁听到 |
|------|--------|
| `{melee}` 或 `{近战}` | 所有近战专精 |
| `{ranged}` 或 `{远程}` | 所有远程专精 |

示例：`{time:00:45} {近战}远离Boss`

### 指定小队

| 写法 | 谁听到 |
|------|--------|
| `{g1}` | 第 1 小队 |
| `{g2}` | 第 2 小队 |
| ... | ... |
| `{g8}` | 第 8 小队 |

示例：`{time:01:00} {g1}去左边 {g2}去右边`

### 组合条件

**逗号 = 或（OR）**，满足任意一个就能听到：

```
{time:00:20} {tank,g1}开怪    ← 坦克 或 第1小队的人都能听到
```

**加号 = 且（AND）**，必须同时满足才能听到：

```
{time:00:10} {治疗+德鲁伊}开宁静    ← 必须同时是治疗专精 且 德鲁伊职业
{time:00:20} {治疗+武僧}开复苏之雾  ← 必须同时是治疗专精 且 武僧职业
```

**混合写法**，加号优先级高于逗号：

```
{time:00:30} {坦克+g1,治疗}注意    ← (坦克 且 第1小队) 或 全部治疗
```

| 写法 | 语义 |
|------|------|
| `{治疗,德鲁伊}` | 治疗 或 德鲁伊 |
| `{治疗+德鲁伊}` | 治疗 且 德鲁伊 |
| `{神圣+牧师}` | 神圣专精 且 牧师职业（= 神圣牧师） |
| `{治疗+牧师+戒律}` | 治疗职责 且 牧师 且 戒律专精 |
| `{坦克+g1,治疗}` | (坦克 且 1队) 或 治疗 |
| `{远程+近战}` | 不可能同时满足，无人听到（不报错） |

## 法术名自动转换

写 `{spell:法术ID}` 会自动变成法术的名字，方便你不用手打技能名：

```
{time:00:03} {奶德1} {spell:2825}     ← 自动变成"英勇"
{time:00:13} {奶德1} {spell:33891}    ← 自动变成"生命之树"
{time:00:27} {奶德1} {spell:740}      ← 自动变成"宁静"
```

法术 ID 怎么查？去 https://www.wowhead.com/ 搜技能，网址里的数字就是 ID。
比如 `wowhead.com/spell=740` 里的 `740` 就是宁静的法术 ID。

注意：`{spell:法术ID}`、`{time:...}`、`{谁}` 这类 token 必须成对闭合；少写 `}` 时，STT 会报出行号并停止本次播报启动。右侧方案文本区行首会显示行号，报错时可以直接对照定位。

## 一行里给不同人说不同的话

同一行可以写多个 `{谁}`，每个人只听到自己那段：

```
{time:00:32} 亡者吐息1:{所有人}被点名恐惧去左边头前 {萨满1}风行
```

| 玩家 | 听到什么 |
|------|---------|
| 所有人 | "被点名恐惧去左边头前" |
| 萨满1 | "风行" |
| — | "亡者吐息1:" 没人听到（这是备注标签） |

### 行内倒数 `{ct:N}`

如果一个机制需要读秒，可以在同一行加 `{ct:N}`：

```
{time:00:10}{ct:5} {所有人}分散
```

`N` 可以写 `1` 到 `10`。上面的例子会在 4.9 秒、5.9 秒、6.9 秒、7.9 秒、8.9 秒依次播放 5、4、3、2、1 的倒数音频；正文仍走普通 TTS 路径，在 10 秒机制命中时播报。

带 `{ct:N}` 的行不叠加设置里的“提前播报时间”，倒数时序只由这一行自己的 `N` 决定。设置页可以关闭倒数播报，也可以在“倒数播报”里切换多游戏、简中/繁中/英文、男/女/播报员风格的内置倒数包；下拉项会显示素材目录 ID，方便和 `media\STTaudio\` 下的文件夹对应，并按 5、4、3、2、1 试听 5 秒。预制语音包只覆盖 1-5 秒；6-10 秒如果当前包和默认包都没有对应音频，就静默跳过。没有正文的行只播倒数，机制命中时保持静默。

普通团员默认启用倒数播报；只有在设置里主动关闭后，`{ct:N}` 才会静默。

### 技能持续时长 `{dur:N}`

如果一个技能不是瞬时事件，可以在技能 token 里加 `dur`，让水平视角在图标右侧显示浅蓝色持续条：

```
{time:00:20} {BOSS}{spell:1249265,dur:5.5}
```

规则：

- `dur` 是持续秒数，范围是 `0 < N <= 600`，支持小数。
- `{dur}` 只影响水平视角显示，不发音，也不改变 TTS 播报时间。
- 非法值会按没有持续时长处理；当前赛季内置模板已经预置常用 Boss 技能的 `dur` 标注。

### 分段进度条 `{bar:N[,tick:M]}`

如果一个机制会在固定窗口内按节奏跳伤害，可以在同一行加 `{bar:持续秒数}` 显示视觉进度条。`tick:M` 可选，表示每 M 秒画一段，并在当前段中央显示本段剩余秒数。

```
{time:00:30} {所有人}{bar:30,tick:3}阶段伤害
```

常用写法：

```
{time:00:08.2,p2} {所有人}{bar:38.5,tick:3.5,spell:1246709}
{time:00:30} {所有人}{bar:30,tick:3,spell:1246709,label:<奶骑吃黄色！>}
{time:01:45} {所有人}{bar:24,tick:2.5,icon:Interface\\Icons\\spell_holy_holybolt,label:<萨哈达尔尖刺>}
{time:02:10} {坦克}抗这一脚{bar:8}
{time:03:00} {所有人}{bar:15,tick:4}
{time:04:00} {bar:38.5,spell:1246709,tick:3.5}
```

规则：

- `bar` 后第一个数字是持续秒数，范围是 `0 < N <= 600`，支持小数。
- `tick` 必须写成命名参数，例如 `{bar:30,tick:3}`；`{bar:30,3}` 无效。
- `spell` 会自动取法术图标和默认文本；`icon` 会覆盖图标；`label:<...>` 会覆盖下方文本。
- 如果这一行除了 `{谁}` 只有 `{bar}`，实时战术板会显示 `label:<...>`；没有 `label` 时显示 `spell` 对应技能名。
- 受众过滤沿用同一行 `{谁}`：写 `{坦克}{bar:30,tick:3}` 时，只有命中坦克条件的玩家看到这条进度条。
- `{bar}` 只做视觉，不发音；需要声音倒数时同一行再写 `{ct:5}`。
- 普通团员默认启用分段进度条；这个功能不需要打开战术方案编辑器。若名单正确但看不到条，先确认 **设置 → 分段进度条** 已启用。
- 位置调整：打开暴雪编辑模式（`/editmode`），拖动名为“分段进度条”的 STT 锚点。

### 行内音效 `{@...}`

如果需要在某个时间点播放音效，可以在同一行加 `{@音频文件}`：

```
{time:00:10}{所有人}{@ding.ogg}分散
```

短文件名会从 `Interface\AddOns\ShengTangTools\media\STTaudio\` 读取；内置默认音效使用 `ding.ogg`，上面的例子会同时播放这个音效和正文“分散”的普通 TTS。也可以写完整插件路径，例如 `{@Interface\AddOns\OtherAddon\media\alert.ogg}`。自定义音频建议使用 WoW 稳定支持的 `.ogg` 或 `.mp3`。没有正文时只播放音效，不触发 TTS。

### 视觉画板 `{board:画板名}`

如果你在“视觉画板”页签画好了一张站位图，可以把它绑到某条时间轴上，触发时自动弹出来播放：

```
{time:00:30,p2} {所有人}{board:P2分散}分散站位
{time:01:10} {所有人}{board:集合点@3}
```

- **按画板名引用**（不是内部 ID）：在某条时间轴文本里嵌入 `{board:你的画板名}`，当这条时间轴触发时就播放该画板。引用的依据是你给画板起的**名字**，不是后台 ID。
- **`@秒`（可选）**：从画板自己的幻灯片时间轴的第几秒开始播，默认 `0` = 从头。例 `{board:P2分散@3}` 表示跳到画板内部第 3 秒再开始播。
- 画板必须**本地存在**才能播。团长广播方案时，STT 会按当前方案的 Boss 打包该 Boss 下全部画板并同步给团员；团员接收后只覆盖这个 Boss 的画板集合，其它 Boss 的画板不受影响。
- `{board:...}` 只负责弹画板，不会进入 TTS 文本——上面例子里 TTS 只念“分散站位”，不会把画板名念出来。

> **用法提示**：在视觉画板页签点“生成模板”会生成当前 Boss 的一组内置模板画板（P1/P2/P3 多张站位图）。引用前确认画板名稳定好记（如 `P3-1左右分组`），再在方案里用 `{board:P3-1左右分组}` 引用。名字改了就要同步改方案里的 `{board:...}`，否则按名字找不到画板，这条引用会静默跳过。

## 人员映射（推荐用法）

在方案顶部加一个 `[人员]` 区块，给角色起别名。这样换人只需改一处：

```
[人员]
坦克1=张三
奶德1=小明
萨满1=老王

{time:00:03} - {奶德1} {spell:2825}
{time:00:13} - {奶德1} {spell:33891}
{time:00:27} - {奶德1} {spell:740}
{time:00:32} 亡者吐息1:{所有人}去左边 {萨满1}风行
```

这里 `{奶德1}` 会自动替换成 `{小明}`，只有小明能听到对应内容。

### 三种映射语义：单人 / 主备 / 全员

`[人员]` 段的等号右侧支持三种写法，根据分隔符自动切换语义：

```
[人员]
坦克1 = 张三              # 1) 单人：{坦克1} → {张三}
奶德1 = 小明, 备奶          # 2) 逗号 = 主备 fallback：小明在场用小明，小明不在用备奶
左边  = 张三 小明 老王       # 3) 空格 = 全员匹配：列出的所有玩家都听到
```

适用场景：

- **单人**：一个角色任务只有一人承担（坦克、特定 cd 圈点）。
- **逗号 fallback**：方案里某个槽位有主备替补；STT 按团内"找到第一个在场的就用，后面忽略"——主奶临时缺席也不需要改方案。
- **空格全员**：分边、分组、分波等"一群人共同承担同一任务"的场景。例：

```
[人员]
左边 = 张三 小明
右边 = 老王 小李

[时间轴]
{time:00:50}{左边} 合圈
{time:00:50}{右边} 接圈
{time:01:20}{所有人} 收尾   # 双边都该听 = 全团听 = {所有人}，不绕分边
```

战术里只要有"分边/分组/分波"的需求，都用空格列名单即可，团长改一行所有人同步生效，不需要每人去 GUI 选自己站哪边。

水平视角会保留空格分组左侧的组名作为对象行，例如正文写 `{种子}` 且 `[人员]` 里写 `种子=DZ1 DZ2` 时，左侧解析区显示 `种子` 一行，不会额外补出 `DZ1`、`DZ2` 两条空行；如果正文直接写 `{DZ1}`，仍会按 `DZ1` 的人员映射显示真实角色。

> 注意：跟 `[打断]` 段空格分隔多名字写法保持一致；昵称（`/st nick`）请避免含空格。团员设置过昵称后，版本检测会用“昵称（原角色ID）”显示，方便团长核对。

#### 嵌套引用：换人只改一处的进阶用法

`[人员]` 支持任意层嵌套展开。配合「导入团员」自动生成的专精槽位，可以写出"换人只改一处"的极简方案：

```
[人员]
DKT1 = 豆豆       # 战术编辑器顶部「导入团员」自动生成
DKT2 = 豆豆2
HD1  = 小明       # 治疗1
HD2  = 小李

左边 = DKT1 HD1   # 引用专精槽位，不写真名
右边 = DKT2 HD2

[时间轴]
{time:00:50}{左边} 合圈
{time:00:50}{右边} 接圈
```

实战时 `{左边}` 先展开为 `{DKT1}{HD1}`，再各自展开为 `{豆豆}{小明}`，走 `T.ShouldBroadcastForNames` 全员匹配。

换人时只改一行 `DKT1 = 新DK`，所有用到 `{DKT1}` 和 `{左边}` 的地方自动同步——这就是 STT「换人只改一处」哲学的真正落地。

引擎对嵌套深度有 5 层兜底（实际很难超过 2 层）；循环引用（如 `左边 = 左边`）会在第二轮自然停止，不会死循环。

### 用内置方案做分边播报（以"鲁拉"P3 为例）


默认情况下，`{左边}/{右边}` 行因为没有玩家映射会**静默不播**——这是有意的"骨架"行为

新安装或恢复默认设置后，每个 Boss 的团队方案默认是空的，不会自动写入内置模板。需要使用内置方案时，请先在战术方案页选择对应 Boss，再点顶部 **从模板重载**。

启用步骤：

1. 战术方案编辑器顶部，把内置方案"复制为个人方案"或"导入到团队方案"
2. 在方案顶部的 `[人员]` 段加两行（注意空格分隔）：

```
[人员]
左边 = 张三 小明 大龙
右边 = 老王 小李 阿二
```

3. 实战 P3：本机角色按所属边听对应的接圈/点符文/星辰提醒，对面边的提醒不会打扰你
4. 团长换人：改一行 `[人员]`，所有 `{左边}/{右边}` 自动同步，不用编辑时间轴

进阶用法：

- 想"双修" → 把自己同时写进 `左边` 和 `右边`：`左边 = 自己 别人 / 右边 = 自己 其他人`
- 想完全关掉分边播报 → 不写 `[人员]` 或 `[人员]` 里不写 `左边/右边` 这两个 key
- P3 末尾的"星辰收尾"是全团机制，写的是 `{所有人}`，跟你选哪边无关

### 使用「导入团员」自动生成

## 阶段相对时间的可用范围

阶段相对时间只对 STT 已内置、并经过验证的 Boss 自动生效。当前宇宙之冕、鲁拉、奇美鲁斯、贝洛朗等已有内置阶段识别，可以直接写 `{time:00:30,p2}`、`{time:00:20,p4}`、`{time:00:12,p1r2}`。

未内置 Boss 不再提供“按技能 ID 自行补阶段规则”的通用降级写法。12.0 以后插件能稳定拿到的是官方 Encounter Timeline / Warning 事件，不是任意 Boss 施法流；很多转阶段技能不会以可安全匹配的 spellID 暴露给插件。旧方案里遗留的 `[设置]` 技能锚点不作为公开推荐用法，能否命中取决于官方时间轴是否暴露对应事件，不承诺可用。

- 线性阶段 Boss 使用 `p1`、`i1`、`p2`、`p3` 这类标签；轮换阶段 Boss 才需要 `p1r2`、`p2r2` 这种“第 N 轮阶段”格式；请使用**小写**。
- 阶段相对时间只影响 `{time:分:秒,阶段}`，不影响普通 `{time:分:秒}`。
- 如果聊天提示“阶段 X 未检测到，N 条事件等待中”，说明自动阶段识别没接上；需要临时推进时，用 `/st phase p2` 这类命令手动切阶段。

### 一个位置多个候选人

如果某个位置经常换人（比如同一个人有好几个号），用逗号隔开：

```
[人员]
奶德1=小明主号,小明小号,小明三号
```

插件会自动找第一个当前在团队里的人。小明主号在就用主号，主号不在就找小号，以此类推。

### 从 lorrgs.io 粘贴

lorrgs.io 导出的格式可以直接用，只需在顶部加 `[人员]`：

```
[人员]
奶德1=你的角色名

{time:00:03} - {奶德1} {spell:2825}
{time:00:13} - {奶德1} {spell:33891}
{time:00:27} - {奶德1} {spell:740}
{time:00:53} - {奶德1} {spell:29166}
{time:01:29} - {奶德1} {spell:47788}
```

中间的 ` - ` 不需要删，它在 `{奶德1}` 前面，自动算备注不播报。

### 使用战术板翻译器转换 NSRT 格式

战术方案编辑器只解析 STT 格式。NSRT 原文不要直接粘到**团队方案**或**个人方案**里；请先打开 `/st` → **设置** Tab → 左侧导航 **系统 → 战术板翻译器**，把原文粘到左侧，右侧生成 STT 文本后再复制回战术方案。

```
EncounterID:3178;Difficulty:Heroic;Name:Vaelgor & Ezzorak;
ph:1;time:19.9;tag:瑟瑟;spellid:322118;
ph:1;time:35.8;tag:瑟瑟;spellid:115310;
ph:2;time:30;tag:tank;text:抗伤;spellid:322118;
```

翻译器识别条件很简单：**第一行必须包含 `EncounterID:数字`**。只有事件行、没有头部的片段不会按 NSRT 解析。

翻译器会生成等价 STT 时间轴，形如：

```
EncounterID: 3178
{time:00:19,p1} {瑟瑟}{spell:322118}
{time:00:35,p1} {瑟瑟}{spell:115310}
{time:00:30,p2} {坦克}抗伤{spell:322118}
```

`ph` 会转换成 STT 阶段标签，`time` 按该阶段的相对秒解释；没有 `ph` 的普通 `{time:MM:SS}` 才按开怪后的绝对时间解释。多数 Boss 按 `ph:N -> pN` 转换；贝洛朗 `EncounterID:3182` 例外，按循环阶段转换为 `ph:1 -> p1r1`、`ph:2 -> p2r1`、`ph:3 -> p1r2`、`ph:4 -> p2r2`。

生成的 STT 文本才是战术方案运行时读取的内容；运行链路不会再自动探测或透明转换外部格式。

**字段对应关系**：

| NSRT 字段 | 翻译结果 |
|---|---|
| `ph:N` + `time:X.Y` | `{time:MM:SS,pN}`（秒数向下取整；贝洛朗按 `p1r1/p2r1/p1r2/p2r2` 映射） |
| `tag:everyone` | `{所有人}` |
| `tag:tank` / `healer` / `dps` / `melee` / `ranged` | `{坦克}` / `{治疗}` / `{输出}` / `{melee}` / `{ranged}` |
| `tag:groupN` | `{gN}` |
| `tag:角色名` | `{角色名}` |
| `tag:瑟瑟 森林`（空格分隔多人）| `{瑟瑟,森林}`（逗号=或） |
| `text:xxx` | 原文透传（含 `{rt1}~{rt8}` 团队标记） |
| `text` 和 `spellid` 都有 | `{谁}文字{spell:ID}` |
| 只有 `spellid` | `{谁}{spell:ID}` |
| `dur` / `countdown` / `TTSTimer` / `sound` / `glowunit` / `bossSpell` / `colors` | 丢弃（STT 一行只对应一个触发点） |

状态栏会实时显示「已解析 N 个事件 / Phase 数: M / 跳过 K 行」，格式不对的行会被自动跳过。

### 使用 MRT 明文时间轴

STT 的**战术板翻译器**也支持 MRT 明文时间轴。打开 `/st` → **设置** Tab → 左侧导航 **系统 → 战术板翻译器**，输入格式选 **MRT**，再按需要选择 Boss，用来把 MRT 的 `pgN` 阶段展开成 STT 阶段标签。

```
{time:0:14,pg2} 集合 |cff33937f玩家|r {spell:370553:24}
{time:0:30,SCC:17:2,pg3} 测试
```

会转换为：

```
{time:0:14,p2} 集合 {玩家} {spell:370553}<扭转天平>
{time:0:30,p3} {所有人}测试
```

规则很少：`pgN` 默认直译成 `pN`；选择级进型 Boss 时按内置 `phaseOrder` 展开，例如 `pg1..pg5 -> p1/i1/p2/i2/p3`；选择轮换型 Boss 时按轮次展开，例如 `pg1/pg2/pg3/pg4 -> p1r1/p2r1/p1r2/p2r2`。MRT 的着色玩家名会在原位置转换成 STT 受众，例如 `|cff33937f玩家|r` 变 `{玩家}`；`{spell:ID}` 后紧跟的技能名会转成备注，例如 `{spell:370553} 扭转天平` 变 `{spell:370553}<扭转天平>`；正文开头没有受众的行才会在 time 后补 `{所有人}`，后续出现的玩家受众不阻止前面普通说明补全员。`SCC/SCS/SAA/SAR`、`wa`、`glow/all` 等 STT 不支持的触发段会静默丢弃，`{spell:ID:大小}` 会保留为 `{spell:ID}`。

### 个人方案：直接粘贴，不需要名字

如果你只想给自己用，可以把 lorrgs.io 的内容直接粘到**个人方案**里，连名字都不用写：

```
{time:00:16} -  {spell:359816}
{time:01:15} -  {spell:363534}
{time:02:25} -  {spell:359816}
{time:04:18} -  {spell:359816}
```

个人方案里没有 `{谁}` 的行会自动对你触发，` - ` 分隔符会被自动清理，播报时只念技能名。

> **注意**：这条只限个人方案。团队方案里没有 `{谁}` 的行照样不播报。

如果你想在个人方案里给不同角色写不同的内容（比如一套方案给治疗号用、一套给坦克号用），仍然可以加条件：

```
{time:08:38} -  {恩护}{spell:363534}
{time:08:38} -  {治疗+德鲁伊}{spell:363534}
```

带了 `{谁}` 的行照常按条件过滤，跟以前一样。

### 个人方案如何覆盖团队方案

当解析方案选择 **团队+个人** 时，STT 默认开启 **个人方案覆盖团队方案（按技能ID）**。

规则很简单：

- 只要个人方案里有一条当前角色真的会听到的同技能播报，团队方案里命中你的那一段同技能内容就会被移除。
- 同一行里其他不是给你的共享提示会保留，不会因为个人方案命中就整行消失。
- 覆盖粒度是 **spellID**，不是第几次，也不是时间窗口。
- 只影响同技能的重复播报；个人方案里没命中你、或根本没有这个技能时，团队方案保持原样。
- 如果你确实想保留旧行为，可以去 **设置 → 方案解析样式 → 方案解析**，关闭 **个人方案覆盖团队方案（按技能ID）**。

例子：

```text
团队方案：{time:00:20} {治疗}{spell:740}
个人方案：{time:00:20} - {spell:740}
```

在 **团队+个人** 模式下，当前治疗自己最终只会听到一条 `740` 播报；如果团队行里还带了给全体治疗的共享提示，那部分仍会保留，不会跟着一起被吃掉。

## 不播报的行（注释）

没有 `{谁}` 的行不会播报任何内容，也**不会显示在左侧时间轴面板**上。可以当注释用：

```
{time:00:00} === P1 开始 ===
{time:00:10} {所有人}全员集合
{time:01:00} === P2 转阶段 ===
{time:01:05} {坦克}拉Boss到中间
```

上面的 `=== P1 开始 ===` 和 `=== P2 转阶段 ===` 没有 `{谁}`，不会播报，
也不会在时间轴面板上显示。它们只存在于右侧的文本编辑器中，方便你看结构。

> 面板上只出现有 `{谁}` 的行；没有 `{谁}` 的行是纯注释，只在编辑器里能看到。没有 `{time:XX}` 的行同样被忽略。

## 颜色文本

你可以用 WoW 标准颜色代码给文字上色：

```
{time:01:52} {spell:1246749} {小野}|cff3fc7eb祝福|r
```

`|cff3fc7eb` 是颜色开头（`3fc7eb` 是十六进制颜色值），`|r` 是颜色结束。中间的"祝福"会在时间轴面板和屏幕提示上显示为对应颜色。

TTS 语音播报时颜色代码会被自动剥离，只念纯文本"祝福"。

常用颜色示例：

| 写法 | 效果 |
|------|------|
| `\|cffff0000红色文字\|r` | 红色（#FF0000） |
| `\|cff00ff00绿色文字\|r` | 绿色（#00FF00） |
| `\|cff3fc7eb青色文字\|r` | 青色（#3FC7EB） |
| `\|cffffcc00金色文字\|r` | 金色（#FFCC00） |

> **格式固定：** `|cff` + 6 位十六进制颜色 + 文字 + `|r`。`cff` 中的 `ff` 是透明度（固定写 ff 即可），后面 6 位才是颜色。

## 注释与静默标记

两种包裹符号的用途不同，别混用：

| 写法 | 含义 | 语音 | 左侧解析区 |
|------|------|------|------|
| `<xxx>` | **绝对注释**：写给自己看的备忘 | 不播报 | **不显示** |
| `~~xxx~~` | **静默标记**：想让团员看到、但不想念 | 不播报 | 显示 `xxx` |

例子：

```
{time:00:10} {所有人}分散 ~~看P1站位图~~       <- 屏幕看得到"看P1站位图"，语音不会念
{time:00:20} {坦克}换坦 <tank2记得给减伤>       <- 只在编辑器里看得到，不播报、也不会显示
{time:00:30} {治疗}{spell:740}<宁静>          <- <宁静> 只是给你自己认 ID 用的注释，完全不影响播报和显示
```

> **旧版迁移**：旧版 STT 中 `<xxx>` 与 `~~xxx~~` 都是"显示但不播报"。从当前版本起 `<xxx>` 的语义改为绝对注释。若你之前依赖 `<xxx>` 显示文字，改成 `~~xxx~~` 即可。

## 智能替换建议

自己写方案时不必记技能 ID：在方案里直接打中文技能名（如"福音"）。左侧解析表格里，凡是含可改写技能名的行，cell 右上角会显示一个**黄色感叹号**。

点击感叹号，cell 下方弹菜单：

```
可改写（1）
  福音  →  {spell:1233865}<福音>    [接受]
                              [忽略]
```

- `[接受]`：只改这一条，源文本里对应行的"福音"改写成 `{spell:1233865}<福音>`。你可以一条一条地决定是否接受。
- 同一行里若有多条（比如"福音"和"宁静"），菜单会列出每条独立的 `[接受]`，底部多一个 `[全部接受此行]`。
- `[忽略]` 或按 Esc 关闭菜单。

替换后 `<福音>` 里的原文保留着——它是绝对注释（见上一节），既不影响播报也不影响渲染，纯粹让你一眼认得这是"福音"。

字典从内置方案、玩家历史方案里出现过的 `{spell:ID}` 自动反查 API 构造，再加上公共光环/团队增益等数据表里的 spellID，无需手动维护。同名多个技能按出现频率取首选；不满意就忽略、自己手动写 ID。

## 自定义配音（语音包系统）

STT 支持语音包：播报文本精确匹配到语音包目录内的 mp3 文件名时，播放真人配音替代系统 TTS。

原理很简单：`{spell:ID}` 在解析阶段会被替换成技能名，到播报时文本已经是"宁静"而不是"{spell:740}"了。STT 拿这个文本去当前语音包目录下找同名 mp3，找到就播，找不到就走 TTS。内置默认语音包统一放在 `Interface\AddOns\ShengTangTools\media\STTaudio\Voice\`，中文文件名只保留在这个目录里。

```
{time:01:30} {治疗}{spell:740}
```

上面这行：`{spell:740}` 解析后变成"宁静" → 匹配到语音包内 `宁静.mp3` → 播放真人配音。直接写 `宁静` 两个字效果完全一样。

### 使用语音包

1. 安装语音包插件（和装普通插件一样，丢到 AddOns 目录）
2. `/reload`
3. 设置 → 语音 → 自定义配音 → 下拉选择语音包

STT 出厂自带默认语音包（宁静、升腾、回溯等治疗大招），开箱即用；这些中文 mp3 都在 `media\STTaudio\Voice\` 下，`media\STTaudio\` 根目录只放内置英文/数字音效。

### 制作自己的语音包

语音包就是一个独立插件，只需要 TOC 文件 + mp3 音频，不需要写任何代码：

```
STT_VoicePack_SeSe/
├── STT_VoicePack_SeSe.toc
└── Voice/
    ├── 宁静.mp3
    ├── 升腾.mp3
    └── ...
```

TOC 文件内容（完整示例）：

```
## Interface: 120000
## Title: STT语音包 - 瑟瑟
## Notes: STT 自定义配音语音包
## Dependencies: ShengTangTools
## X-STT-VoicePack-Name: 瑟瑟语音包
## X-STT-VoicePack-Path: Interface\AddOns\STT_VoicePack_SeSe\Voice\
```

mp3 文件名必须和游戏内技能名一模一样。建议像上面一样统一放进语音包插件自己的 `Voice\` 子目录，避免中文配音和插件内置音效混在一起。放到 AddOns 目录后 `/reload`，设置里就能看到你的语音包了。

没匹配到配音的文本正常走 TTS，不受影响。

## 常见问题

**Q：我写了一行但没有播报？**
检查两点：1) 是否有 `{time:XX}`；2) 是否有 `{谁}` 指定了受众。两个都要有才会播报。

**Q：怎么让所有人都听到？**
用 `{所有人}` 或 `{所有人}`。团队方案中不能省略——不写 `{谁}` 等于注释。个人方案中不写 `{谁}` 会自动对你触发。

**Q：`{time:00:30,p2}` 这种写法支持吗？**
支持。表示“P2 开始后第 30 秒播报”。宇宙之冕、鲁拉、奇美鲁斯等已有内置阶段识别，直接写即可，不需要 `[设置]`。

**Q：`p1r2` 是什么意思？**
“第 2 轮进入 P1”。`{time:00:12,p1r2}` = 第二次进 P1 后 12 秒播报。只有奇美鲁斯、贝洛朗、双龙这类反复回到同一阶段的 Boss 才需要写 `rN`；鲁拉、宇宙之冕这类线性阶段 Boss 直接写 `p1`、`p2`、`p4`。阶段标签用小写，`P1R2` 无效。

**Q：`[设置]` 是必须的吗？**
不是。已内置阶段识别的 Boss 不需要写。只有未内置的 Boss 想用阶段相对时间才需要补。普通 `{time:分:秒}` 不受影响。

**Q：STT 能不能像 DynamicCam 一样把镜头拉得更远？**
可以。打开 **设置 → 界面 → 超远视距**，开总开关后拖动 **最大镜头距离** 滑块（CVar `cameraDistanceMaxZoomFactor`，面板范围 15–39，默认 39，对应游戏内 2.6 倍距离）。范围完全对齐 DynamicCam 的标准设置。每次客户端登录 STT 会按你的设置自动写回这项 CVar，不需要其它插件。**调整后请手动用鼠标滚轮往外缩小视角，才能看到新的最大距离生效**。下方有一个 **重置** 按钮，可一键回到游戏默认值。关闭总开关只停止下次写入，不会自动把已修改的视距还原。

**Q：自我位置标记可以换成纯色块吗？**
可以。打开 **设置 → 界面 → 自我位置标记**，把 **材质** 选成 **纯色块** 即可。纯色块可以自定义颜色，也可以开关描边；尺寸默认 `16`，可在 `1-100` 间调整，透明度、动效、仅战斗中显示和位置拖动仍沿用原来的设置。

**Q：贝洛朗光环提醒还需要开 `/st debug` 吗？**
不需要。它已经是正式功能，默认关闭；在设置页开启 **贝洛朗光环提醒** 后，战斗中会显示蓝/黄光环图标，提示框缩放最低可调到 `0.10`。需要调整位置时，点 **解锁提示框位置** 后拖动 STT 金色框体，调好再点 **锁定提示框位置**；**重置位置** 只会恢复默认位置，不会改你的缩放和透明度。脉冲动画可保持循环，也可以勾选 **仅在获得/刷新光环时播放一次脉冲**。需要声音时，再单独开启 **蓝/黄羽毛音频播报**。内置羽毛音效使用 `media\STTaudio\beloren_yellow.ogg` 和 `media\STTaudio\beloren_blue.ogg`。

**Q：个人光环提醒怎么测试鲁拉星辰裂片？**
它已经是正式功能，默认关闭。确认 **设置 → 屏幕提醒** 里已有可用样式后，到 **设置 → 个人光环提醒** 打开开关。设置页按 **BOSS 分组** 管理个人提醒；组内每行只显示 **启用 / 光环名称 / 对应屏幕提醒样式 / 测试 / 编辑**，玩家日常只需要选择样式、测试和开关。默认内置鲁拉 **星辰裂片** 预设；未校准的内置预设会标记为 **实验**，但仍可自行启用。点击 **编辑** 会打开原有规则编辑弹窗，里面可调整 Boss、难度、severity、时间窗口、显示名称、倒计时、倒数语音、是否仅匹配应播放声音和屏幕提醒样式；时间窗口可写 `0:05-1:30, 2:10-2:40`，留空表示全程匹配。点击某个 BOSS 组里的 **新建自定义** 会自动继承该 BOSS；底部 **新增 BOSS 分组** 可输入 Boss ID，也可输入中文名从现有 BOSS 映射里匹配。鲁拉 M 难度 P4 中，默认星辰裂片提醒命中时还会用 `文本#1` 额外叠加 `← / → / ←` 方向箭头（实验）；它只在 P4 的 13、33、53、73、93 秒三连方向窗口内显示，窗口外只保留原星辰裂片提醒。单条规则的 **测试** 按钮不受时间窗口和 shouldPlaySound 限制，方便战斗外调样式。团长或助理可以 **Shift+点击自定义规则** 只下发这一条，也可以 **Shift+点击左侧个人光环提醒模块名** 下发整页规则；接收方选择 **合并 / 替换 / 拒绝**，合并会追加规则，替换会按 Boss、难度、severity、时间窗口和显示名称覆盖同规则并保留其他本地规则。点 **观测记录** 可查看最近一场战斗中 STT 收到的个人警告，列表会按 Boss、难度、severity 和公开名称/SpellID 去重。这个功能只响应暴雪 `ENCOUNTER_WARNING`，不读取 private aura 或隐藏光环 ID。

**Q：团本指挥面板怎么启用？**
它已经从 Beta 模式移出，默认关闭；可在设置页启用，或用 **`/st rcp`** 开关。面板有 4 个可单独开关的子功能：战复次数和充能统计、自己身上的嗜血/精疲力尽 CD、战斗计时、死亡事件列表。觉得窗口太小的话，可在设置页调整 **团本指挥面板显示比例**，战复/嗜血/计时和死亡列表会一起放大或缩小。死亡事件默认不播放任何音效或 TTS；如果你需要语音提醒，可在设置页单独开启死亡 TTS，默认最多播报前 2 个死亡，之后静默，默认格式是 **`{sourceName}死了`**，也可以改成自己的文案。**死亡详情（实验）** 可在普通设置页直接开启；启用后点击死亡行会打开二级窗口，按时间线显示暴雪 Death Recap 能提供的伤害/死亡信息，并合并 STT 在死亡前记录到的治疗、吸收和血量上升兜底事件；鼠标悬停技能行可查看对应技能提示；如果暴雪没有生成回放且 STT 也没有最近事件，会明确显示无可回放数据。

**Q：打断轮替怎么用？**
在团队方案里写独立 `[打断]` 块并同步即可。**打断轮替** 和对应 Boss 开关默认启用；没有收到 `[打断]` 块时不会显示、播报或推进轮替。默认样式是经典横幅；如果想看纵向顺位和读条进度，可把 **打断提醒样式** 切到 **卡片栈**。当前样式的显示位置可在设置页点 **解锁显示位置** 后拖动，调好后点 **锁定显示位置**。

**Q：怎么把 STT 调试日志发给开发者？**
先执行 **`/stt debug`** 开启关键诊断采集，复现问题后输入 **`/stt log`**。STT 会打开一个可复制窗口，点进文本框后全选复制即可。这个日志主要记录异常、用户操作、实战关键事件和通信结果，不再保留大量启动/注册流水；当前会话最近 1000 条会被保留，`/reload` 后会清空。

**Q：怎么查看懒加载模块和内存变化？**
用 **`/st mem`** 查看严格懒加载内存快照、模块 desired/loaded/pending 状态和 GC 回收量。用 **`/st perf`** 查看性能快照；需要对比时，先用 **`/st perf baseline`** 记录基线，再用 **`/st perf diff`** 看当前差值。用 **`/st mod`** 或 **`/st modules`** 查看模块状态；执行 **`/st mod enable 模块名`** 或 **`/st mod disable 模块名`** 只写入配置并提示 `/reload`，彻底加载或卸载以重载后的 `/st mem` 为准。

**Q：自动战斗日志怎么开？**
在设置页打开 **自动战斗日志**。它会按当前区域类型和难度决定是否开启 `/combatlog`：默认记录英雄/史诗团本和大秘境；普通团本、随机团本、普通/英雄/史诗 5 人本默认关闭，需要时可以单独勾选。开启 **自动启用高级战斗日志** 后，STT 在需要开始记录时会把 `advancedCombatLogging` 设为开启。进副本、换区域或钥石开始后会延迟重新评估，设置页的 **立即重新评估** 可以按当前区域立刻修正状态。

**Q：怀疑 STT 导致卡顿时，怎么把性能日志发给开发者？**
先执行 **`/st plog on`** 开启性能采样，复现卡顿后输入 **`/st plog`**。STT 会打开一个可复制窗口，内容是最近的性能 CSV，包括周期采样、慢操作、战斗/遭遇状态和阶段切换。复制完成后用 **`/st plog off`** 停止采样；这个采样默认关闭，且不会自动开启 `addOnProfilerEnabled`。

**Q：可以用 MRT 的 `{H}治疗内容{/H}` 吗？**
不支持这种块级写法，会被自动清理。请用 `{治疗}治疗内容` 代替。

**Q：旧方案用的 `{{双括号}}` 还能用吗？**
能用，会自动转换成单括号。但建议新方案直接用单括号 `{}`。

**Q：中文标点可以用吗？**
可以。`{time：01：30}` 里的中文冒号 `：` 会自动转成英文 `:`。中文花括号 `｛｝` 和全角加号 `＋` 也会自动转换。

**Q：`{治疗+德鲁伊}` 和 `{治疗}{德鲁伊}` 有什么区别？**
`{治疗+德鲁伊}` 是 AND，必须同时满足两个条件。`{治疗}{德鲁伊}` 是两个独立花括号，等于 OR（治疗或德鲁伊都能听到）。

**Q：`{神圣}` 会匹配两个职业？**
对。神圣牧师和神圣圣骑都叫"神圣"，所以 `{神圣}` 两个都会听到。加职业名区分：`{神圣+牧师}` 只匹配神圣牧师，`{神圣+圣骑士}` 只匹配神圣圣骑。防护、冰霜、恢复同理。

**Q：`{BOSS}` 和 `{所有人}` 有什么区别？**
`{所有人}` 全团都会听到语音播报。`{BOSS}` 不会播报给任何人，只在时间轴面板上显示 Boss 技能时间点。

**Q：我的方案没有 `{BOSS}`，时间轴面板上看不到 Boss 技能了？**
对。面板只显示有 `{谁}` 的行，想看 Boss 技能时间线就加 `{BOSS}`。内置模板已自带。

**Q：没有 `{谁}` 的注释行为什么不显示了？**
面板只显示带 `{谁}` 的行。注释行仍在编辑器中，面板不显示。

## 工作区布局

- 战术方案页右下角支持自由缩放；放大窗口只会增加可视区域，不会改字号、行高、列宽和单元格大小。
- 战术方案默认打开水平视角，左上角“视角”仍可切换到垂直视角。垂直视角适合逐行检查文本，水平视角按“对象 × 时间”展示技能点，Boss、职责、专精和玩家各占一行；行头会尽量显示 Boss 头像、职责图标、专精图标或玩家专精图标。
- 水平视角的对象行默认按方案源文本里首次出现的顺序固定。拖动技能点不会因为某一行的最早技能时间变化而自动重排对象行。需要整理时，点击左上角“对象”列头会按对象整理一次当前显示顺序；这只影响当前显示，不会改右侧方案文本。右键左侧对象首列的列头、人员行头或空白首列区域，都可以打开轨道管理并新增人员行；保存后会写入 `[人员]`。只填行名会写成 `行名` 占位行，填写映射才写成 `行名=玩家名`。如果行名没有命中默认专精黑话，弹窗里选择的专精会写入 `[人员图标]` 作为水平视图显示提示。双击正文 `{谁}` 派生出的玩家行或已有 `[人员]` 行的行头，都会打开人员行编辑器；如果重命名正文派生行，STT 会同步改写正文里的精确 `{旧名}` / `{{旧名}}` 目标。
- 水平视角里，带 `{dur:N}` 的技能点会在图标右侧延伸浅蓝色持续条，用来扫技能从开始到结束的时间窗口。
- 水平视角里，同一对象下时间相同或视觉上会重叠的技能点会自动展开到下一行；后续展开行不重复对象名，方便直接扫完整时间线。
- 水平视角的对象行右端有一个加号按钮：点开后对象行的时间轴会留空，下面会按技能拆出若干子行，每个 spellID 占一行，方便单独检查某个技能的节奏。再点一次回到合并显示。展开状态会随设置保存，`/reload` 后仍然保留；只放过一种技能的对象行不会出现这个按钮。
- 水平视角里，单击技能点会选中并高亮该技能点，同时定位右侧源文本。`Ctrl/Command+点击` 可以多选或反选技能点，`Shift+点击` 会从当前主选中技能点范围选择到目标技能点；点击行只聚焦该行，`Ctrl/Command+点击` 行头会选中本行技能点；双击可编辑的人员行头会打开人员行编辑器。
- 水平视角里，直接按住技能点拖动可以改写右侧源文本的 `{time:...}`；拖到另一条对象行时，也会把该技能点的 `{谁}` 改成目标行。拖动未选中的技能点会先选中它；拖动已选中的任意技能点会批量移动整个选区。拖动超过 5px 才会进入拖拽状态，避免误触普通点击。拖到临近技能点旁边时会临时换行避让，拖走后空行会自动收敛。
- 水平视角里，可以在某个对象行的时间轴轨道内右键。右键轨道或技能点会先把播放头移动到鼠标位置；右键菜单顶部会显示对象名、鼠标位置时间和更新后的播放头时间；右键技能点时第一项是“编辑”，也可以直接双击技能点打开事件编辑弹窗。弹窗会把当前 DSL 行投影成时间、相位、目标、技能、文本、倒数音效、自定义 TTS 提前、屏幕提醒提前、持续、进度条、音效、团队标记和备注字段，底部只读源文本实时预览保存后的整行内容；右侧 MRT/STT 文本仍然是唯一权威，保存一次就是一格撤销记录。其它主动作是“在此插入技能”“粘贴到此处”“粘贴到播放头”“移动选中到此处”“移动选中到播放头”“预览此时间点”，并保留“剪切/复制/删除”。右键已框选技能点时操作当前选区；右键未框选技能点时操作命中的单个技能点；右键空白轨道时主要用于插入、粘贴和预览。点“在此插入技能”会打开左侧技能抽屉；抽屉会按玩家职业、Boss 或“全部”自动选分类，也支持按技能名、spellID、英文或拼音搜索；技能类 Tooltip 会显示技能说明和技能ID，单击技能卡会立刻写回右侧文本，例如 `{time:01:30} {瑟维莱}{spell:363534,dur:4}<回溯>`。在阶段相对的 Boss 水平视图中插入 Boss 技能时，会默认写成阶段内时间，例如 `{time:00:30,p2}`，而不是开怪后的绝对秒。右侧 MRT/STT 文本仍然是唯一权威，插入后可以继续手动编辑或用撤销回退。
- 水平视角空白处按住左键拖拽可以框选多个技能点；起点不要求在某一行内，从时间轴空白背景开始也可以。框选过程中会显示半透明金色矩形，松手后被覆盖的技能点保持高亮；按住 `Ctrl/Command` 框选会把框内技能点追加到当前选区或从当前选区反选。随后可以右键执行复制、剪切或删除，也可以用 `Ctrl/Command+C`、`Ctrl/Command+X`、`Ctrl/Command+V` 操作选区。`Ctrl/Command+A` 会先选中当前行技能点，再按一次选中当前方案技能点；`Delete/Backspace` 删除选区，`Esc` 清空选区，方向键切换主选中技能点。多技能点粘贴和批量拖动都会保留它们之间的相对时间；跨职业粘贴不做合法性检查，spellID 原样写入。
- 拖动时始终显示垂直对齐线和当前目标时间；默认按当前缩放自动选择网格，低/中缩放仍按 1 秒移动，放大后可细到 0.5/0.2/0.1 秒。按住 `Shift` 拖动可临时关闭网格，做自由移动。释放鼠标后，原本的时间格式会保留，例如 `{time:1:30}` 会继续写成分秒格式，`{time:90}` 会继续写成纯秒格式；拖到小数秒时会写成 `{time:1:30.5}` 或 `{time:90.5}`。
- 水平视角顶部播放控制器从左到右是播放/暂停、时间码、停止。鼠标悬停在水平时间轴内时，也可以按 `空格` 播放或暂停。暂停会保留播放头位置，再次播放从当前位置继续；停止会结束当前播放并回到起点。
- 顶部播放头可以拖动定位；单击时间尺空白处会跳转到该时间点，并保持原来的播放/暂停状态。播放时播放头接近右侧边界会自动横向跟随；如果你手动滚动底部滚动条、`Shift+滚轮` 或中键拖动，自动跟随会暂停到下一次播放。
- `Ctrl+Z` / `Ctrl+Shift+Z` / `Ctrl+Y` 是战术编辑器级撤销/重做；Mac 下也支持 `Command+Z` / `Command+Shift+Z` / `Command+Y`。两种视角都生效，撤销记录只保留当前打开会话，不写入配置。这些编辑快捷键在战术方案窗口内会被 STT 消费，不会继续触发游戏快捷键。
- 水平视角专用快捷键：`Shift+滚轮` 横向移动，`Option/Alt+Shift+滚轮` 以鼠标所在时间点为中心缩放；顶部时间尺会随着缩放自动切换 60/30/10/5/1/0.5 秒等刻度精度。缩到最小时会按当前方案最长时间纵览全局，也支持中键拖动和底部滚动条横向移动。这些水平视角鼠标操作只作用于 STT 时间轴，不会触发游戏信号或鼠标/滚轮绑定。
- 水平视角底部会常驻一条低对比度灰字快捷提示；鼠标移入时间轴时会提亮，点击提示可展开全部水平视角快捷操作。实际横移、缩放、拖动、定位、粘贴、删除、批量移动、撤销或重做后，灰字会短暂切换成当前操作反馈，例如“已粘贴 3 个技能点到 1:23.5 · Ctrl/Command+Z 可撤销”，再自动回到默认提示。
- 左侧解析区和右侧文本区之间新增可拖拽分隔线；拖动时会实时调整两侧宽度比例，双击可回到当前视角的默认比例。
- 窗口大小、位置、分隔线比例和上次关闭时所在标签页都会自动记忆；设置页中的“重置战术方案布局”可恢复默认布局。
- 设置页支持 **系统 → 系统设置 → 设置面板字体大小**，会同步放大字号、控件和导航宽度；同页的 **入口显示** 可单独开关小地图图标，以及是否显示在暴雪 **选项 → 插件** 列表。只有战术方案页支持右下角拖拽改变窗口大小。
- 窗体外观可以在 **设置 → 系统 → 窗体外观** 切换。它只影响 STT 设置主窗的边框、背景和标题条，预设包含基瑞安、温西尔、通灵领主和法夜；战斗 HUD 与实时战术板不跟随这个设置。
- 实时战术板的“显示样式”可以在“经典”“聚焦”“简洁”之间切换。设置页会先显示通用选项，再按当前样式只显示对应的专属选项：通用显示里的“显示播报对象”只控制每条事件前面的 `{谁}` 名字是否显示，不改变“显示全部事件”的范围，也不影响 TTS；“法术显示”可选“图标+文本”“仅图标”“仅文本”，其中“仅图标”只隐藏 `{spell}` 解析出的技能名，保留后面的动作说明；经典样式可以自定义当前条目的颜色、不透明度、材质、左侧高亮条宽度和像素发光参数；列表样式设置还包含背景透明度、显示头部、自动恢复、滚动速度、过期事件、当前事件位置、时间方向和远期隐藏；聚焦显示设置包含上方邻居数、下方邻居数、聚焦强度、聚焦宽度、聚焦行距、聚焦停留、聚焦离场动效和对齐方式。切到聚焦时会自动关闭头部并把背景透明度设为 0%，切回经典或简洁后不会自动恢复旧值，需要时可手动再调。
- 经典样式保持原来的滚动列表；简洁样式使用更紧凑的单行文本时间轴，玩家名按职业色显示，技能图标内嵌在文字里，并继续使用你选择的倒计时格式、倒数位置、自动滚动、过期事件和远期隐藏设置；聚焦样式会把当前条目固定在中央放大高亮，如果下一条还没到点会按“聚焦停留”短暂停留，再平滑滑向下一条；如果下一条已经到点，则直接追上去，不把每条都强制延后一秒。上方显示刚过去的条目，下方显示即将到来的条目。聚焦样式使用独立的上下邻居窗口和柔边聚焦背景，不继承列表模式的左侧指示条、过期事件、远期隐藏或时间方向设置。聚焦强度会同时协调中心大小、邻居层次和透明度；聚焦行距只调整各行之间的空隙；聚焦离场动效只在当前条目倒数归零后的停留窗口内，让固定背景后的内容队列先慢后快地向上滑入下一条。如果想整体放大或缩小战术板，请使用面板缩放。倒数位置对三种样式都生效，可选“右侧”或“图标前”。实时战术板解锁后可以继续横向拉大到接近屏幕宽度，方便显示长技能名和长说明。
- 实时战术板的“条目样式”可以在“清爽（默认）”和“卡片”之间切换；显示全部事件时，清爽样式会去掉单元格深色背景，只保留并排文本和原有悬停交互。
- 实时战术板设置里新增的 **战斗外常驻显示** 开关默认关闭；勾选后，战斗外会按当前选中 Boss 方案铺出完整时间轴预览（时间静态不递减），切换 Boss 自动刷新预览，进入战斗自动让位给真正的实时模式，战斗结束后再自动回到预览。
### 视觉画板：像 PPT 一样画站位

主界面“视觉画板”页签可以画 Boss 站位示意图，并把它绑到时间轴某个时刻自动播放。新建画板会按当前 Boss 写入默认场地背景。

**四种组件，从抽屉拖进画布**

画布左侧是“组件抽屉”，把组件拖到画布上即可（顶部不再有一个个的“添加”按钮）。只有 4 种组件：

- **人员控件**：一个三合一的复合控件，专精图标 + 浅绿散圈 + 名签捆在一起，整体拖动、整体选中。它对接方案里的 `[人员]` 槽位，编辑时显示槽位名（如 `咕咕2`），运行时自动显示真实角色名。
- **文本**：自由文字，比如门口、集合点。
- **形状**：矩形、圆、线、箭头，画十字参考线、分边框这类。
- **团队标记**：暴雪 8 个团队标记（星、圆、钻、三角、月、方、叉、骷髅）。

**人员预设：照着 `[人员]` 自动给你摆好**

抽屉里的人员预设会按当前激活方案的 `[人员]` 段自动生成——方案里有 `咕咕2=咕咕玩家` 这样的单条映射，抽屉里就会出现一个对应预设，拖进画布即用。运行时画板会把槽位名换成真实角色名显示；如果某个人就是你自己，会自动把你的位置高亮出来（高亮形态可以自定义，比如放大、描边、发光）。没有激活方案或方案没写 `[人员]` 时，预设区会提示你先选战术方案，你仍可以拖一个空白人员控件进画布自己填槽位名。

**幻灯片帧动画：画两张图，系统帮你补间**

视觉画板像做 PPT 一样工作：底部帧条可以新增“图1、图2……”多帧。你只要把图1摆好，再画一张图2把要移动的人挪到新位置，运行时播放时系统会自动平滑补间——两帧都在的人员会从旧位置平移到新位置，只在后一帧才出现的人员会淡入登场。每帧可以单独设停留时长和补间时长。编辑器里有“预览播放”，不进战斗就能看图1→图2的过渡效果。

**无边画布：随便缩放、随便拖**

画布是无边的：滚轮缩放、中键或空格+左键拖动平移；画布中央有一圈“画板框”表示真正会播放的范围，框外是随便摆放的草稿区。拖动组件时会自动出现对齐参考线并吸附到其它组件或画板中线，对齐站位很省心。

**专精图标选择器**：人员控件默认按 `[人员]` 映射的职业专精自动显示图标，不用手动指定；想换成别的图标时，在属性区点“换图标”弹出图标选择器，顶部搜索框可按职业名或专精名即时筛选，下方按职业分组铺出全专精图标网格。图标数据按当前版本的游戏接口动态生成，换版本不需要手动维护。

**形状的 PPT 式缩放**：选中矩形等形状后，拖动四周句柄会像 PowerPoint 那样锚定对边/对角——拖右边只往右长、左边不动；拖角点固定对角，宽高一起变。圆、图标、人员控件按中心径向缩放；文字按句柄竖直距离调字号；线/箭头拖端点改朝向。用上方句柄旋转。

**多选与成组**：按住 Shift 左键可把多个组件加入或移出选区；框选范围内的组件会一起被选中。`Ctrl+G` 成组，`Ctrl+Shift+G` 解组。点组内任一组件会选中整组，对组拖动即整体移动。所有移动、缩放、成组、解组都进撤销栈，`Ctrl+Z` / `Ctrl+Shift+Z` 可回退重做。

**图层面板**：编辑器右侧的图层面板像 Figma 一样自顶到底列出所有组和顶层组件，组可折叠/展开。点击某行在画布上选中并定位对应组件，选中始终稳定对应、不会串位；双击行名改名；眼睛切换显示/隐藏，锁切换锁定；拖动行重排层级。组头还提供**批量调整**：点组头的批量按钮，可以对整组人员一次性统一改同一项属性（比如统一把散圈调大、统一改名签字号），一步到位、一次撤销。

**绑定到时间轴播放**：战术文本里写 `{board:画板名}` 会在该时间点播放画板，`{board:画板名@5}` 会从画板内部第 5 秒开始播放；引用按画板的**名字**匹配，不是内部 ID，画板 token 不会进入 TTS 文本。同名画板优先在当前 Boss 下匹配，避免不同 Boss 的同名画板串台。广播语义 Boss 方案时，STT 会把当前 Boss 下全部本地画板一起打包下发，团员接收方案时只覆盖该 Boss 的画板集合。详见上文「视觉画板 `{board:画板名}`」一节。

**Boss 画板导入导出**：在“视觉画板”页选择任意一张画板后，点“导出Boss”会导出这张画板所属 Boss 下的全部画板；点“导入Boss”并粘贴 `STT-VBOARD-BOSS:1:` 字符串后，会自动切到包内 Boss，并用包内画板覆盖本地同 Boss 的画板集合。导入不会合并，也不会碰其它 Boss。

## 方案同步权限

- **战术方案核心** 是 STT 基础能力，默认可用：可以接收团长推送的团队方案、解析方案并在战斗中按分配播报。
- **战术方案编辑器** 采用按需懒加载。只有切到战术方案页时，才会加载导入、编辑、水平视角、技能抽屉、撤销等编辑工具；团员不写方案、不打开该页时，不会加载编辑器界面。
- STT 默认只接收团长推送的团队方案，避免随机团员误同步覆盖正经方案。
- 如果团长临时需要接收团员推送，可以在 **设置 → 方案解析样式 → 方案同步** 关闭“仅接收团长的方案推送”。
- 接收端成功应用同步方案后，会用屏幕黄字提示收到的方案名和发送者；如果同步内容包含视觉画板，也会在同一条提示里显示新增、更新、跳过数量，并向发送端回传同步确认。
- 发送端会根据团员回传确认汇总失败结果：有人明确拒收时提示对应成员和原因；有人没有返回确认时提示“未返回同步确认”。不会仅凭 STT 版本号差异报警。
- 非团长发起同步时，发送端会提示“开启仅接收团长推送的团员会拒收”；接收端因该限制拒收时，会显示屏幕提示并写入 STT 普通日志，同时向发送端回传拒收原因。
- 关闭后会恢复为接受所有团队成员推送；个人方案仍不参与同步。



## 触发轴（技能触发播报）

> 💡 GUI 中悬停对应控件可查看触发轴、受众、技能 token 和屏幕提醒提前量的 Tooltip 解释；按住 Alt 展开示例。

大秘境没有固定时间轴，适合用"技能触发"来决定什么时候播报。团本里如果某个 Boss 的节奏更适合按技能事件写，也可以直接用触发轴。

### 核心区别

| 维度 | 团本时间轴 | 触发轴 |
|------|-----------|-------------|
| 行头 | `{time:分:秒}` | `{on:spell:技能ID}` |
| 触发方式 | 开战后第 N 秒 | Boss 释放指定技能时 |
| 适用场景 | 团本固定时间点 | 大秘境、团本事件点 |
| 其余规则 | `{谁}` + 内容 | 完全一样 |

**只有行头不同，其余写法和团本一模一样。**

### 格式

```
{on:spell:技能ID} {谁}播报内容
{on:spell:技能ID:-N} {谁}提前N秒播报内容
{on:spell:技能ID:-0} {谁}按触发点播报内容
{on:spell:技能ID}#N {谁}第N次的播报内容
{on:spell:技能ID:-N}#N {谁}第N次提前N秒播报内容
```

- `{on:spell:技能ID}` — 触发标识，表示"当这个技能触发时"。**不会被播报**，只是告诉插件什么时候播。
- `:-N` — 可选，表示在技能真正触发前 N 秒就先播报一次。
- `:-0` — 可选，表示这一行不继承全局提前秒数，按技能触发点准时播报。
- `#N` — 可选，指定第 N 次触发才播报。
- `{谁}` — 和团本一样，没有 `{谁}` 就不播报。

### 对照速查

| 写法 | 播报效果 |
|------|----------|
| `{on:spell:466064} {所有人}躲正面` | 全员听到"躲正面" |
| `{on:spell:466064} {坦克}嘲讽` | 坦克听到"嘲讽" |
| `{on:spell:466064} 备注:{所有人}快躲` | 全员听到"快躲"（"备注:"不播报） |
| `{on:spell:466064}#2 {所有人}第二次集合` | 第2次触发播"第二次集合" |
| `{on:spell:466064:-15} {治疗}预铺HOT` | 技能前15秒对治疗播"预铺HOT" |
| `{on:spell:466064:-0} {所有人}准时躲正面` | 覆盖全局提前设置，技能触发时播"准时躲正面" |
| `{on:spell:466064:-5}#2 {所有人}第二次还有5秒` | 第2次触发前5秒预警 |
| `{on:spell:466064} {所有人}{spell:466064}快躲` | 全员听到"技能名 快躲" |
| `{on:spell:466064}` | 不播报（纯触发标记） |
| `{on:spell:466064} 这是备注` | 不播报（无 `{谁}`） |

### 提前预警

`:-N` 适合需要提前准备的场景，比如治疗预铺、减伤预读、换位预备。

- `{on:spell:466064:-15} {治疗}预铺`：技能前 15 秒提醒治疗准备。
- `{on:spell:466064} {所有人}注意大伤害`：技能真正触发时再补一句正式播报。
- 同一个技能可以同时写 advance 和 normal 规则，互不覆盖。
- 剩余时间不足 N 秒时，插件会立即播这条 advance 规则，不会丢掉。
- 没写 `:-N` 的行会继承设置里的”提前 X 秒播报”；全局提前时间可在 0 到 10 秒内按 0.1 秒调整。写了 `:-N` 以行内值为准，不叠加全局秒数；写 `:-0` 就是本行强制准时播报。

### 屏幕提醒指示器

屏幕提醒现在是“指示器列表”：时间轴命中一条提醒时，所有启用的指示器都会各显示一份。指示器不绑定具体战术行，也不按 Boss、职业或副本过滤；要不要显示、提前几秒显示、显示成什么样，都在 **设置 → 屏幕提醒** 里配置。

- 默认有 1 条文本指示器，显示本次提醒文字和倒数。
- 可新增 4 类指示器：文本、图标、计时条、环形。
- 屏幕提醒页顶部的 **全局提前量** 默认是 3 秒。指示器的提前量模式只有两种：**使用全局提前量** 或 **自定义提前量**。使用全局时，未写 `{sr:N}` 的文本按全局提前量显示，写了 `{sr:N}` 就按这条文本的 N 秒显示；自定义时，这个指示器只按自己的提前量显示，忽略全局和 `{sr:N}`。
- 倒数不是模板字符串，而是结构化选项：字号、位置、小数位、单位、包裹符号、随时间变色。
- 文本指示器可以在“技能显示”里选择 `{spell:ID}` 的显示方式：仅文本、仅图标、图标+文本。比如 `{所有人}{spell:740}` 可以显示为“宁静”、宁静图标，或“宁静图标 宁静”；普通文字会保留，只替换其中的技能 token。
- 文本、图标、计时条可以设置多实例生长方向：向上、向下、向左或向右；图标指示器支持旧版时钟形 CD 转盘；文本指示器可设置描边颜色；文本、图标、计时条都可以按需启用“发光像素”边框。该效果默认关闭，启用后可选择自定义时长，或跟随本条指示器的“延后显示”时长，同时可调整线条和粒子、频率、长度、粗细和偏移。延后显示是否淡出可独立开关，顶部预览会按当前提前量、延后显示和淡出设置播放。
- 文本来源始终是本次时间轴提醒内容；指示器只负责样式，不改写播报文案。
- 列表里的对钩表示这条指示器已启用；选中指示器后，可在右侧设置顶部用“启用”复选框开关它。
- 解锁锚点后，设置主窗会收缩成左上角的屏幕提醒编辑工具条，各指示器锚点淡入显示；可以拖动每条启用指示器的位置，单击某个锚点会在旁边打开这条指示器的参数面板，点空白处关闭参数面板。工具条里可直接测试当前样式，也可点“锁定”回到完整设置窗，位置和参数会保留到下次 `/reload`。

### 屏幕提醒提前量 `{sr:N}`

如果某一条战术文本想让屏幕提醒更早或更晚出现，可以在同一行写 `{sr:N}`：

```
{time:01:00} {所有人}{sr:10}集合
```

这行对“使用全局提前量”的指示器会提前 10 秒显示；没有 `{sr:N}` 时使用屏幕提醒页顶部的全局提前量，默认 3 秒。`{sr:N}` 只影响屏幕提醒，不影响 TTS、倒数 `{ct:N}`、持续 `{dur:N}` 或进度条 `{bar:...}`。如果某个指示器选择了“自定义提前量”，它始终按自己的提前量显示，不接受全局值，也不接受 `{sr:N}`。

### 指示器路由 `{to:名称}`

默认屏幕提醒会提供 `文本#1`、`计时条#1`、`环形#1` 三条样式，其中 `计时条#1` 和 `环形#1` 默认勾选“不参与通用文本提醒”。如果某条屏幕提醒只想给一个指定指示器显示，在 **设置 → 屏幕提醒** 里勾选这条指示器的“不参与通用文本提醒”，然后在战术行里写 `{to:指示器名称}`。新建指示器会自动得到无空格名称，例如 `文本#1`、`环形#1`。
在 chip 编辑器里也可以用“屏幕提醒”下拉选择指示器；保存后的效果等同于手写 `{to:名称}`。如果要调整本条文本的屏幕提醒提前量，用“屏幕提醒提前(sr)”字段，保存后等同于手写 `{sr:N}`。

```
{time:01:00} {to:环形#1} {所有人}集合
```

这行到点时只会触发名叫“环形#1”的指示器；其它广播指示器不会显示。没有 `{to:...}` 的普通战术行仍然只触发未勾选专属模式的指示器，已勾选专属模式的指示器会保持隐藏。

多个目标用半角逗号分隔：

```
{time:01:30} {to:重要提醒,计时条#1} {所有人}分散
```

`名称` 就是列表里的指示器名称；可以在指示器设置里重命名，输入的空白会自动去掉。改名后，战术行里的 `{to:...}` 也要同步改。

### `{on:spell:xxx}` 和 `{spell:xxx}` 的区别

这两个长得很像，但作用完全不同：

- `{on:spell:466064}` — **触发标识**，写在行头，不播报，等于"当技能 466064 触发时"。
- `{spell:466064}` — **技能名替换**，写在内容里，播报时自动变成技能的名字。

### 完整示例

```
[方案]
名称 = 迈萨拉洞窟 - 凡尔克斯
作者 = 玩家

[人员]
坦克1 = 张三

[触发轴]
{on:spell:466064:-15} {治疗}预铺HOT
{on:spell:466064} {所有人}躲正面
{on:spell:466556} {所有人}进罩子
{on:spell:465904} {坦克1}准备大减伤
{on:spell:466556}#2 {所有人}第二次进罩子
{on:spell:466556}#3 {所有人}最后一次进罩子
```

### 团本中使用触发轴

触发轴不限大秘境，团本也能用。某些技能按事件点播报更合适的，直接整份方案写成 `[触发轴]` 就行。固定时间点仍推荐 `[时间轴]`；两种格式二选一，不混写。`#N` 和 `:-N` 同样支持。

### 旧格式兼容

旧的管道符格式 `{spell:xxx}|模式|内容` 仍然可用，插件会自动识别。
但新方案建议统一用 `{on:spell:xxx}` 格式——写法更简单，和团本规则完全一致。

## 鲁拉符文助手

鲁拉（L'ura）Boss 需要按顺序踩 5 个符文。打开 **设置 → 鲁拉符文助手**，直接把 5 个符文按钮和 1 个撤回按钮拖到动作条上就能用；拖拽时 STT 会按需自动创建或更新对应的通用宏。

**屏幕符文按钮条**默认关闭，推荐外场点符文的坦克和 4 个治疗启用。按钮条可以限制为仅在进军奎尔丹纳斯团本内显示；开启这个限制后，还可以再勾选“仅在鲁拉战斗中启用按钮条”，按钮条会按 encounter 3183 鲁拉战斗状态显隐。解锁按钮条后，拖动金色框体可调整位置，拖右下角拉手可调整排列，自动吸附为横排、4+1、3+2、2+2+1 或竖排；新角色和重置后默认位于鲁拉符文面板下方。

**重要：不要屏蔽点符文人员。** 鲁拉符文依赖团队频道实际收到点符文玩家发出的聊天。如果你屏蔽了负责点符文的人，暴雪会直接让你收不到那个人的团队频道消息，STT 也就无法显示那一枚符文。开打前请全团临时取消屏蔽外场点符文人员，尤其是负责点符文的坦克和 4 个治疗；个人恩怨先放一放，过了鲁拉再说。

拖拽时，STT 会创建或更新对应的 6 个**通用宏**：5 个符文宏（`◇STT`、`△STT`、`TSTT`、`○STT`、`XSTT`）和 1 个撤回宏（`撤回STT`）。符文宏默认使用团队频道发送 STT 自带符文贴图路径，例如：

- `◇STT`：`/raid Interface/AddOns/ShengTangTools/media/rune_rhom`
- `△STT`：`/raid Interface/AddOns/ShengTangTools/media/rune_tran`
- `TSTT`：`/raid Interface/AddOns/ShengTangTools/media/rune_t`
- `○STT`：`/raid Interface/AddOns/ShengTangTools/media/rune_circle`
- `XSTT`：`/raid Interface/AddOns/ShengTangTools/media/rune_x`
- `撤回STT`：`/rw 撤回`

“清理符文宏”按钮只删除 STT 生成的这 6 个同名符文宏；如果存在同名宏但正文不是 STT 生成格式，会跳过，避免误删玩家自己改过的宏。

团长在战斗外切换“符文频道”或“符文分配模式”时，STT 会把这两项设置同步给队伍/团队成员；团员本地已有的 STT 符文宏会跟着刷新。这个同步会改频道和分配模式，不会替团员自动创建缺失宏。

内置鲁拉方案会在 M 难度 P1 三轮符文自动播放 5 个纯数字音效：第一轮 33/34/35/36/37 秒播放“1/2/3/4/5”，第二轮 95/96/97/98/99 秒播放“1/2/3/4/5”，第三轮 157/158/159/160/161 秒播放“1/2/3/4/5”，用来提醒负责点宏的人按当前出现的符文。这些提示来自战术方案里的 `{@Announcer_1.ogg}` 到 `{@Announcer_5.ogg}` 行；如果你不想要，可以复制内置方案后删除这些行。普通和英雄难度的符文窗口时间不同，使用这份内置方案时可能有时间偏差。

接收方还可以在 **设置 → 鲁拉符文助手** 打开“符文已点即时音效（推荐点符文团员启用）”。打开后，每次收到指挥点宏的聊天并显示符文图标时，会立即播放“1符文点了”到“5符文点了”的本地 OGG 音效，告诉外场上一个符文已经按下。这个开关默认关闭，不走系统 TTS 队列，和内置方案里的纯数字音效互不替代。

## 鲁拉水晶提醒

如果你负责丢黎明水晶，可以在 **设置 → 鲁拉水晶** 打开“黎明水晶提醒”。这个开关默认关闭；打开后，只有本机玩家在鲁拉战斗中成功释放 `1253050`（黎明水晶）时，STT 才会立即显示本地屏幕提醒倒计时，默认 3 秒、默认使用已有的 `计时条#1`。你可以单独开启或关闭“播放倒数语音”；开启时会复用全局倒数播报语音包播放 3、2、1，效果等价于这条本地提醒自带 `{ct:3}`。

设置页里的“放下黎明水晶后的倒计时（秒）”指水晶落地后的爆炸前窗口；倒计时结束后，黎明水晶开始对全团造成 AOE 伤害，所以需要在倒计时结束前捡起来。你也可以把提醒样式改成 **设置 → 屏幕提醒** 里已经做好的任意样式；如果当前选择的样式不存在，启用或测试时不会自动创建，请先在屏幕提醒中创建或改选样式。

如果符文面板背景板挡视野，可以在 **设置 → 鲁拉符文助手 → 符文面板背景板透明度** 里把背景板透明度调到 0%-100% 之间；符文图标、中央 BOSS 图形和文字不会跟着变透明。

符文面板会在 2、5 号位置显示低透明红色圆形背景，在 3 号位置显示低透明蓝色圆形背景。符文还没出现时，也能先看出红场负责 2/5，蓝场负责 3；M 难度 P4 横排记忆符文不显示这层分场背景。

坦克角色会自动使用与参考面板一致的 180 度翻转视角；M 难度 P4 的横排记忆符文仍保持横排，不随坦克视角翻转。

默认情况下，符文分配模式为“按团长事件分流”：团队领袖消息固定显示到 1、4，普通团队消息显示到 2、3、5。STT 不解析聊天内容，只把收到的符文贴图路径原样放进对应位置。

如果需要更简单的显示，可以在 **设置 → 鲁拉符文助手 → 符文分配模式** 里切到 **按收到顺序**，所有符文聊天会按 1、2、3、4、5 顺序显示。战斗中发送者与 GUID 会被暴雪保护为 secret value，STT 不提供按发送者分配模式。

符文窗口期内，收到 `/rw` 团队警告会直接撤回上一个符文；STT 不读取团队警告正文。设置页里的撤回按钮只是帮你创建并拖拽一个 `/rw 撤回` 宏。因为撤回依赖 `/rw` 团队警告频道，所有需要点撤回宏的治疗都必须有团长或团队助理权限。

### 拖拽到动作条

1. 输入 `/st` 打开设置页
2. 进入 **鲁拉符文助手**
3. 在“符文快捷按钮”区域，把 5 个符文按钮和撤回按钮左键拖到动作条
4. 开打时按踩符文的顺序依次点击；点错时用撤回按钮撤回上一枚

第一次拖拽时会自动创建缺失宏；后面继续拖同一个符文或撤回按钮，会复用同名宏并按当前设置更新，不会重复占宏槽。符文宏正文使用当前符文频道和 STT 自带贴图路径，撤回宏正文固定为 `/rw 撤回`。

## 打断轮替块（`[打断]`）

打断轮替只读取**团队方案**里的独立 `[打断]` 块，不读取个人方案，也不读取外部笔记。团长保存团队方案后，STT 会通过现有方案同步链路把这段内容发给团员；团员收到后会随当前团队方案一起重读。打断轮替和当前 Boss 开关默认启用；如果没有收到 `[打断]` 块，本模块不会显示、播报或推进轮替。

设置页里的 `[打断]` 文本框只是复制范例，不是正式编辑入口；请把示例复制出来，粘贴到**战术方案 → 团队方案**后再按本团名单修改。

写法（下面的 `DK2`、`LR1` 等需要先在 `[人员]` 里定义）：

```text
[打断]
1: DK2 DK1 LR1 DKT1
2: 咕咕2 SS1 FS1 SS2
3: DH1 AM1 LR2 元素1
```

- `[打断]` 独占一行；遇到下一个 `[xxx]` 顶级块或方案结束时自动结束。
- 每行格式是 `序号: 玩家ID或映射名 玩家ID或映射名 ...`，序号后写冒号，成员之间用空格分开；英文冒号 `:` 和中文冒号 `：` 都可以。
- 可以直接写玩家的游戏内 ID、每个人在 `/st nick` 设置的昵称，也可以写 `[人员]` 里的映射名；`DK2`、`LR1` 这类通常就是 `[人员]` 里定义好的职业槽位映射名。映射名右侧可以写真实角色名、昵称或多个逗号候选，STT 会按当前团队里能识别到的第一个候选作为显示名；本机识别自己时会同时匹配映射名、映射值和自己的昵称。
- `[打断]` 的顺位严格按文本顺序计算，不会因为某人暂时不在团队、死亡或冷却而跳过；映射名会优先按 `[人员]` 解析，未定义别名则按原文当作玩家名处理。
- 首版只在 M 难度鲁拉启用，按 boss 开关显示为“至暗之夜”。鲁拉这里需要 3 行，分别对应 `boss2`、`boss3`、`boss4`；如果后续 Boss 有多组打断，也按行继续写 `2:`、`3:`。
- **设置 → 打断轮替 → 鲁拉打断宏（实验）** 会显示当前职业打断技能图标，直接把这个图标拖到动作条即可生成/更新一个通用宏。团队方案保存或同步后，STT 会按你在 `[打断]` 里的位置自动切换“1/2/3 号打断”和“第几断”，并刷新已有通用宏；没有同步方案或临时换人时，也可以手动改两个下拉。
- 鲁拉打断宏的 1/2/3 号打断分别对应 `boss2`、`boss3`、`boss4`。第 1-3 断只打自己的 Boss；第 4 断会从当前 Boss 向前回退，例如 3 号第 4 断会生成 `[@boss4,harm,nodead][@boss3,harm,nodead][@boss2,harm,nodead]`。
- 轮替严格按名单线性循环，不会因为死亡或技能冷却跳过某个人。
- 鲁拉打断宏仍是实验功能，宏未经过实战测试，不保真能使用；设置页拖拽按钮 tooltip 也会提示这一点。
- 默认显示方式是**经典横幅**。
- 如果想看纵向顺位和读条进度，可以在 **设置 → 打断轮替 → 打断提醒样式** 切到 **卡片栈**：轮替名单纵向排列，自己的卡片固定金黄高亮，中央选择器对准当前读条对应的顺位，下面的进度条显示 Boss 读条进度。**BOSS 施法条放大提醒** 只在当前 Boss 开关启用且对应 encounter 激活时显示。
- 当前打断提醒样式的显示位置可在设置页点 **解锁显示位置** 后拖动 STT 金色框体，调好后点 **锁定显示位置**；**重置显示位置** 只重置当前样式主体，不影响 BOSS 施法条放大提醒。
- 经典横幅的屏幕提示驻留时长和文本横幅缩放可在设置页调整；轮到自己时播放的音效也可自定义，但音效文件必须放在 `ShengTangTools/media/STTaudio/` 下。音效输入框留空时使用内置 `interrupt.ogg`。

`[设置]` 是给阶段锚点等系统参数用的；`[打断]` 是给实战打断名单用的，两者互不替代。


## 施法记录

点击“收集团队施法记录”后，打开战术方案「水平视角」甘特图即可查看团队施法记录。甘特图会按团员显示多条同款只读施法记录行，和方案技能点上下对照，复盘“计划与全团实际”的差距。

- **团队收集**：点击“收集团队施法记录”会刷新当前 Boss 的团队施法记录，并在战术方案甘特图里显示。
- **甘特图临时行**：打开战术方案「水平视角」，甘特图最上方会出现只读的施法记录行。它的行首是角色名（用职业色显示）、左侧有青色竖条、整行青色调背景 —— 即使方案里也安排了同名角色，也能一眼区分。它显示与当前 Boss 匹配的最近一场录像里的全部施法，每个技能图标套青色边框；收到团队广播后会按团员显示多条同款行，自己在最上方，其余按名字排序。这些行不能拖拽、不能编辑、不进撤销，纯展示。
- **对照查看**：临时行的施法图标和下方方案技能点用同一时间刻度。甘特图缩放（Alt+Shift+滚轮）和横向滚动（Shift+滚轮）时，施法图标跟随移动。一场战斗施法很多、图标会密集重叠，放大即可看清局部。
- **蓄力法术**：成功释放会显示为一个带持续时间的施法记录；取消或失败会显示红色边框与红色持续条。
- **显示开关**：编辑方案时若不想看这条行，可在 设置 → 战术方案 → 施法记录（实验） 关闭“在战术方案甘特图显示施法记录行”。
- **保留场次**：默认只保留最近 1 场，可在设置里调整（1–5）；超出后最旧的录像会被自动删除。
- **独立回放窗口**：设置页提供“打开回放窗口”按钮，可以打开一个能选历史多场、带播放控制和技能筛选的独立窗口。

施法记录是纯复盘工具，只做战后呈现，不在战斗中驱动任何操作；手动收集时只发送本场施法时间点、技能 ID、蓄力持续时间和失败标记。


## 团长设置下发

团长或团队助理可以在 STT 设置页把本机设置下发给队伍或团队成员。单项设置用 **Shift+点击** 对应控件；整页设置用 **Shift+点击左侧模块名**。普通点击仍然只修改本机设置。

在 **设置 → 屏幕提醒** 的指示器列表里，**Shift+点击某一条样式** 会只下发这一条样式；**Shift+点击左侧屏幕提醒模块名** 会下发整页屏幕提醒。接收方会看到 **合并 / 替换 / 拒绝**：合并是增量追加，同名样式会自动生成 `名称-2`、`名称-3` 这类副本名；替换会覆盖同名样式并追加新样式，但保留接收方本地其他无关样式。整页替换会同步团长的全局提前量等模块级设置。

在 **设置 → 个人光环提醒** 的 BOSS 分组里，**Shift+点击某一条自定义规则** 会只下发这一条规则；**Shift+点击左侧个人光环提醒模块名** 会下发整页规则。接收方同样会看到 **合并 / 替换 / 拒绝**：合并是增量追加规则；替换会按 Boss、难度、severity、时间窗口和显示名称匹配同规则，命中则覆盖，没命中则追加，并保留接收方其他本地规则。规则里的屏幕提醒样式按名称引用，样式本体仍由屏幕提醒同步或导入导出单独管理。

屏幕提醒页底部的 **导入配置 / 导出配置** 只管理屏幕提醒配置，导出的字符串以 `STT:SR:1:` 开头，不会进入全局设置导入导出，也不会影响其他 STT 设置。

接收方默认开启接受。收到下发后会弹出确认框，只有点击“接受”或列表导入的“合并 / 替换”才会应用；也可以点“拒绝”、按 Esc，或勾选“以后不再接受该发送者”。被忽略的发送者可以在 **设置 → 屏幕提醒** 点击“重置下发黑名单”清空，也可以用 `/st pushreset` 清空。

接收方可在 **设置 → 系统 → 团长设置下发** 关闭“接受团长下发设置”。关闭后不会弹窗，也不会应用任何下发设置。

出于安全和本地差异考虑，STT 不会下发调试开关、TTS 语音 ID、视觉画板本地素材路径、接受开关本身，以及没有 `dbPath` 的自定义控件。跨版本遇到未知设置项时，弹窗里会显示为未识别项，但不会应用。



## 配置管理

入口：**战术方案页顶部第二行的“配置”下拉**。

配置是一份独立的方案库。每个角色首次登录时会自动拥有一份自己的配置；下拉里也能看到其它角色创建过的配置，名称会按创建者职业着色。

- **切换配置**：直接在“配置”下拉里选择目标配置。切换不会重载界面，当前方案、解析区和时间轴会刷新到新配置；每个角色会记住自己上次选择的配置，`/reload` 后不会被切回私有配置。
- **新建配置**：在下拉末尾点“+ 新建配置...”，填写名字，可选择复制当前配置、空白创建，或从其它配置复制内容。
- **复制内容**：在下拉末尾点“⇩ 从其它配置复制内容...”，选择来源后确认，会把来源配置的全部方案覆盖到当前配置。
- **编辑配置**：在下拉末尾点“✎ 编辑当前配置...”，可修改名字；删除时需要再点一次确认。删除当前配置会先自动切到其它配置；如果只剩这一份，会自动新建一份空配置。
- **设为新角色默认**：在下拉末尾点“★ 设为新角色默认（账号共享）”，当前配置会标记为 `★默认`。之后首次登录的新角色仍会自动创建自己的空配置，但会先激活这份默认配置，适合团长多号共用同一套方案。
- **取消默认配置**：当前配置已经是默认时，再点“✓ 已设为新角色默认（点击取消）”即可取消。已有角色的当前选择不会被改动，只影响未来首次登录的新角色。
- **角色自动配置**：首次登录新角色时，STT 会自动创建“角色名-服务器”配置；如果没有账号默认配置，就直接切到这份私有配置。

旧版方案会在首次加载时迁移到当前角色的“角色名-服务器”配置，原有方案内容保留不变。
